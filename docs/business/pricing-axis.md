# 요금제 구분 축 (Pricing Axis)

> 작성: 서연(PM) · 2026-06-26 · **한나(가격분석가) 입력용**
> 이 문서는 "무엇으로 무료/유료를 가를지"만 정한다. 단위원가·마진·금액은 한나 몫.
> ⚠️ 일부 제품 사실 미확정 → 하단 "도현 확인 필요" 먼저 닫아야 축이 확정된다.

---

## 1. 결론 먼저 — 핵심 축 (1차 = 사용량, 2차 = 좌석)

| | 축 | 역할 |
|---|----|----|
| **주축 (Primary)** | **① 월 회의 처리 건수** | 무료/유료를 가르는 메인 미터. "회의 1건 = 문서 1세트"가 제품의 자연 단위 |
| **보조축 (Secondary)** | **⑤ 동시 협업 좌석(seat)** | Free/Pro(개인) → Team(다인) 점프의 차별점 |
| 가드레일(상한만) | ② STT 분량(분) — *조건부* | 원가 폭주 방지용 **상한**으로만. 메인 미터로는 안 씀 (이유 아래) |

**버린 축**: ③ 문서 종류 제한 / ④ 저장 용량.

---

## 2. 왜 이 축인가 (근거)

### 주축 = 월 회의 건수 (채택)
- **사용자 행동 단위와 일치**: 사용자는 "이번 달에 회의 몇 번 돌렸나"로 가치를 체감한다. 분(min)·문서 개수보다 직관적. → 결제 저항 낮음.
- **실질 원가와 연동**: z.ai는 호출당 과금이 아닌 **구독 정액(쿼터)**이지만, 회의 1건 = 문서 14종 = **19~22분 처리 + 동시성 슬롯 점유**(메모리: doc-generation-architecture). 즉 회의 건수가 곧 **캐파(처리능력) 소모량**. 정액 구독이라도 동시 처리에 상한이 있으므로 건수 = 비용 프록시로 타당.
- **H3 가설과 정합**: "회의 1건 = 문서 1세트가 핵심 사용 단위"가 검증되면 이 축이 정답.

### 보조축 = 좌석(seat) (채택)
- P0(1인 개발자)는 좌석 1로 충분 → 개인 Pro.
- SI·에이전시(P1)는 산출물을 **팀 공유**가 필요 → Team 티어로 좌석·공유 권한 차별화.
- 좌석은 **B2B 객단가를 올리는 정석 축**. 사용량과 직교(orthogonal)라 깔끔하게 결합된다.

### STT 분량(분) = 가드레일로만 (조건부)
- STT는 **유일한 진짜 종량 변동비**(Whisper $0.006/분). → 한 회의가 3시간이면 원가가 튄다.
- 하지만 **메인 미터로 쓰면 안 됨**: (a) 사용자가 "분"으로 가치를 못 느낀다 (b) **현재 브라우저 STT 폴백이면 STT 원가 0일 수 있음**(아래 확인 필요).
- → 결론: "회의 1건당 최대 N분" 같은 **상한 가드레일**로만 둔다. 가격을 가르는 노브는 아님.

### 버린 축 — 왜 안 쓰나
| 버린 축 | 이유 |
|---------|------|
| ③ 문서 종류 제한 (14종 중 몇 개) | **가치 훼손**. H2 가설상 사용자는 14종 중 2~4종만 쓰는데, 그게 사람마다 다르다. 무료에서 종류를 막으면 "내가 필요한 그 문서"가 막혀 가치 체험 자체가 실패 → 전환 안 됨. 무료에선 **다 보여주고**, 건수로 제한하는 게 맞다. |
| ④ 저장 용량 | **원가 미미**(Supabase 무료 1GB, 변환용 임시사본은 STT 후 삭제 — stt-storage 메모리). 사용자도 용량을 가치로 안 느낌. 차별 노브로 약함. |

## 3. 티어 설계 (3단: Free / Pro / Team)

> 금액은 비워둠 — 한나가 원가·마진으로 채운다. 여기선 **차별 노브**만.

| | **Free** (체험·후킹) | **Pro** (개인 P0) | **Team** (P1 SI/에이전시) |
|---|---|---|---|
| **월 회의 건수** | 소량 (예: 3건) | 넉넉 (예: 30건) | 더 많음 + 좌석당 합산 |
| **좌석** | 1 | 1 | 다인 (좌석당 과금) |
| **문서 14종** | **전부 사용 가능** | 전부 | 전부 |
| **내보내기 (Word/Excel/PPT)** | ✅ (후킹 핵심, 막지 말 것) | ✅ | ✅ |
| **회의당 길이 상한** | 짧게 (가드레일) | 길게 | 길게 |
| **공유/협업** | 본인만 | 본인 + 공유링크 | **팀 공유·권한** |
| **보관 기간** | 짧게 (예: 30일) | 무제한 | 무제한 |
| (검토) 우선 처리/속도 | 일반 | 일반 | 우선 |

**티어 간 점프 논리**
- Free → Pro: **회의 건수 부족**으로 막힘 (양적 한계).
- Pro → Team: **여러 명이 같이 써야 함**(좌석·공유) (질적 한계).

## 4. 전환 트리거 (무료 → 유료, 어디서 막히게 할까)

**핵심: 가치는 충분히 맛보게 하되, "한 번 더 돌리려는 순간" 막는다.**

| 순서 | 트리거 | 설계 |
|------|--------|------|
| 1 (메인) | **월 회의 건수 소진** | 무료 N건을 다 쓰고 "이번 달 또 회의를 문서화하려는 순간" 막힘. 이미 가치를 봤기에 결제 동기 최고 |
| 2 | **긴 회의 처리 불가** | 무료 회의당 길이 상한 초과 → 실제 업무 회의(보통 길다)는 Pro 필요 |
| 3 | **함께 쓰려는 순간** | 동료에게 공유·협업하려 할 때 Team |
| 4 | **보관 만료** | 무료 30일 후 과거 회의 잠금 → 자료 누적된 사용자가 Pro로 |

> **막지 말아야 할 것 (전환 망치는 함정)**: 문서 종류·내보내기. 이걸 막으면 가치 체험이 끊겨 **전환은커녕 이탈**한다. 무료의 목적은 "이 제품이 진짜 문서를 만들어주는구나"를 1회라도 끝까지 경험시키는 것.

## 5. 도현(개발팀장) 확인 필요 — *이게 닫혀야 축 확정*

| # | 질문 | 왜 필요한가 (어느 결정에 영향) |
|---|------|------------------------------|
| **D1** | 베타 현재 STT는 **Whisper(종량 $0.006/분)인가, 브라우저 STT(무료) 폴백인가?** 사용자가 고를 수 있나? | STT가 무료면 가드레일 자체가 약화 / 종량이면 길이 상한 필수. 메모리상 .env에서 켜졌다 꺼졌다 했음 |
| **D2** | z.ai 구독은 **월 호출/토큰 쿼터가 정해진 정액인가?** 동시 처리 슬롯의 실질 상한은? | "회의 건수 = 캐파"라는 주축 전제의 근거. 쿼터 한계가 무료 N건 산정의 천장 |
| **D3** | **회의 1건당 LLM 비용이 사실상 균일한가?** (회의 길이/문서 종류에 따라 크게 달라지나) | "회의 건수" 단일 미터가 성립하려면 건당 비용 분산이 작아야 함. 크면 분량 축 비중↑ |
| **D4** | **회의당 처리 가능한 최대 길이(분)**의 기술적 상한은? (Vercel 4.5MB/300초, 청크분할 미구현 — 메모리) | 무료/Pro 길이 상한 숫자의 현실 천장 |
| **D5** | **회의 건수 카운팅·좌석·보관기간 제한을 구현하는 난이도/일정**은? (현재 사용량 추적 테이블 없음 — admin 메모리상 "사용량 추적 후" 단계) | 이 축으로 과금하려면 미터링이 선행. 없으면 1차 출시 축을 단순화해야 함 |
| **D6** | 내보내기(Word/Excel/PPT)에 추가 변동비가 있나? (서버 처리 비용) | "내보내기 무료 개방"이 원가상 안전한지 |

> D5가 특히 중요: **미터링이 없으면 "건수 제한"을 강제할 수 없다.** 그러면 1차는 "좌석/길이"처럼 구현 쉬운 축부터 가고 건수는 미터링 붙은 뒤로 미룰 수도 있다. 도현 답 받고 한나와 재조정.

## 6. 한나에게 넘기는 "확정 축" 요약 (TL;DR)
- **주축**: 월 회의 처리 건수 (= 문서 1세트 단위). 무료/유료 가르는 메인.
- **보조축**: 좌석(seat) — Pro(개인) → Team(다인).
- **가드레일**: STT 분량은 "회의당 최대 길이" 상한으로만(원가 폭주 방지). *D1 결과에 따라 비중 조정.*
- **3티어**: Free(체험·건수소량·14종전부·내보내기개방) / Pro(개인·건수넉넉·길이↑·무제한보관) / Team(좌석·팀공유).
- **전환**: 무료 회의 건수 소진이 메인 트리거. 문서종류·내보내기는 절대 안 막음.
- **미확정**: D1(STT 종량 여부)·D3(건당 비용 균일성)·D5(미터링 구현)이 닫혀야 숫자 산정 가능.
