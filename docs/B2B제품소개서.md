# MeetingAutoDocs — 제품 소개서

> 회의 녹음을 기획 문서 14종으로 자동 변환하는 AI 솔루션

![제품 메인](../public/landing/01-main.png)

---

## 회의는 끝났는데, 문서는 누가 만드나요?

프로젝트는 회의에서 결정되지만, 그 결정을 **PRD·화면설계·DB설계·API명세**로 옮기는 일은
늘 누군가의 야근으로 남습니다. MeetingAutoDocs는 이 과정을 자동화합니다.

**회의 녹음 한 번 → 기획 문서 14종 자동 생성.**

---

## 이렇게 동작합니다

| 단계 | 내용 |
|------|------|
| ① 입력 | 브라우저에서 녹음하거나, 회의 녹음 파일을 업로드 |
| ② 회의록 | 음성을 텍스트로 변환하고 AI가 개요·핵심 논의·결정·할 일로 정리 |
| ③ 문서 생성 | 요약을 바탕으로 문서 14종을 의존성 순서대로 자동 작성 |
| ④ 내보내기 | Word·Excel·PowerPoint·Markdown으로 즉시 공유 |

## 도입하면 이렇게 달라집니다

| Before | After |
|--------|-------|
| 회의 후 기획자가 며칠간 문서 작성 | 회의 직후 문서 14종 자동 확보 |
| 결정 사항 문서화 누락 → 개발 단계 재논의 | 회의록·문서로 결정이 그대로 보존 |
| 기획 인력 없으면 문서 없이 개발 | 기획 인력 없이도 표준 산출물 확보 |
| 문서가 흩어지고 버전이 어긋남 | 의존성 자동 전파로 문서 일관성 유지 |

## 핵심 기능

- **문서 14종 일괄 생성** — 기획부터 배포까지 한 흐름으로
- **의존성 자동 전파** — 상위 문서를 고치면 하위 문서에 반영
- **4종 포맷 내보내기** — Word(DOCX)·Excel(XLSX)·PowerPoint(PPTX)·Markdown
- **시각화 내장** — 플로우차트·ERD·화면 다이어그램을 문서 안에서 렌더링
- **계정 단위 데이터 격리** — 문서는 사용자 계정에만 안전하게 저장
- **설치 불필요** — 웹 브라우저 하나로 바로 사용

## 이런 팀에 적합합니다

- 기획 인력이 부족한 **스타트업·소규모 개발팀**
- 다수 프로젝트의 산출물을 빠르게 표준화해야 하는 **SI/에이전시**
- 회의는 많지만 문서화에 시간을 뺏기는 **모든 프로젝트 조직**

---

## 도입 절차

1. **문의** — 팀 규모·사용 목적 공유
2. **시연/PoC** — 실제 회의 녹음으로 산출물 확인
3. **도입** — 계정 발급 후 바로 사용
