## 0. 계산 단위와 핵심 가정

| 항목 | 값 | 근거 |
|------|-----|------|
| **계산 단위** | 회의 **1건** (= 녹음 1개 → 문서 1세트) | 서연 product-definition H3 "회의 1건 = 문서 1세트가 핵심 사용 단위" |
| 회의 길이 | **20분** | 프로젝트 메모리 기준값. (틀리면 STT 원가가 길이에 정비례로 변함 — 4번 민감도) |

## 1. STT (음성 → 텍스트) 단가

STT는 키 유무로 원가가 **0이 되기도, 종량이 되기도** 한다. 그래서 시나리오를 둘로 가른다.

| 분기 | 조건 | 단가 | 출처 |
|------|------|------|------|
| **A. 브라우저 STT** | `OPENAI_API_KEY` 없음 → transformers.js(Xenova/whisper-small) 클라 실행 | **$0 (사용자 기기 연산)** | 실데이터: stt factory.ts 키 분기, 메모리 stt-storage-architecture |
| **B. Whisper 종량** | `OPENAI_API_KEY` 있음 → 서버 Whisper(whisper-1) | **$0.006/분** | 실데이터: OpenAI 공식($0.006/min = $0.36/hr, 2026-06 확인) |

### 회의 1건(20분) STT 원가

| 시나리오 | 계산 | 원가(USD) | 원가(원) |
|---------|------|----------|---------|
| A. 브라우저 STT | $0 | **$0** | **0원** |
| B. Whisper | 20분 × $0.006 | **$0.12** | **약 168원** |

> **참고(더 싼 옵션):** OpenAI는 2026년 `gpt-4o-mini-transcribe`를 **$0.003/분**(절반)으로 제공한다. 코드는 현재 whisper-1 경로($0.006). mini로 바꾸면 B 시나리오 원가가 84원으로 반감 → **도현에게 전환 가능성 확인**.
> **⚠️ 현재 .env 상태:** `.env.local`에 `OPENAI_API_KEY`가 **SET 되어 있음** → 현재 배포는 **B(Whisper 종량) 경로**로 추정됨. "베타는 무료 브라우저STT"라는 가정과 어긋날 수 있음 → **도현 확인 1**.

## 2. LLM (요약 + 문서 14종 생성) 단가 — "정액 ≠ 원가 0"

### 2-1. 회의 1건당 LLM 호출/토큰 소비 (실데이터: 코드 실측)

LLM은 z.ai 정액이라 "시간"은 원가가 아니다. 그러나 **정액엔 한도가 있고, 건당 토큰 소비량이 한도를 갉아먹는다.** 그래서 먼저 건당 소비를 센다.

문서 14종 중 4종(PRD·배포·테스트계획·WBS)은 **섹션 청킹**으로 여러 콜을 친다. 나머지는 단일 콜.

| 단계 | 콜 수 | 출력 max_tokens(콜당) | 근거 |
|------|------|----------------------|------|
| 요약 | 1 | 8,192 | summarize/route.ts |
| PRD (청킹) | **15** | 16,384 | prdSections=15섹션, 동시성3 |
| 배포가이드 (청킹) | **9** | 16,384 | deploymentSections=9 |
| 테스트계획 (청킹) | **9** | 16,384 | testPlanSections=9 |
| WBS (청킹) | **9** | 16,384 | wbsSections=9 |
| 나머지 10종 (단일) | 10 | 8,192~12,288 | api-spec/database 16,384 등 차등 |
| **합계** | **약 53콜** | — | 요약1 + 14종(청킹4종=42콜 + 단일10종=10콜) |

> **핵심:** "14종 = 14콜"이 아니다. 청킹 때문에 **회의 1건 = 약 53 LLM 콜**이다. 정액 한도를 요청 수로 따질 때 이 53이 결정적이다.

**토큰 추정(추정):** 콜당 입력(프롬프트+컨텍스트) 3~6K + 출력 평균 4~8K → 콜당 약 10K 토큰. 53콜 × 10K = **회의 1건당 약 500K~700K 토큰** (입출력 합산, 추정). 정확한 실측은 도현(로그) 확인.

### 2-2. 정액 → 건당 환산원가 (★ 핵심 미지수 — 확인 필요)

정액 환산원가 공식:
```
건당 환산원가 = 월 구독료 ÷ (월 정액 한도 ÷ 건당 소비)
             = 월 구독료 ÷ 월 처리 가능 건수
```

이 공식의 입력값 3개 중 **2개가 코드/메모리에 없다(확인 필요):**

| 입력값 | 현재 아는 것 | 상태 |
|--------|-------------|------|
| 건당 소비 | 약 53콜 / 약 500~700K 토큰 | 실데이터(콜) + 추정(토큰) |
| z.ai 코딩플랜 **월 구독료** | 미상 | **확인 필요(도현)** |
| z.ai 코딩플랜 **월 한도**(요청수/토큰/일prompt) | 미상 | **확인 필요(도현)** |

→ **현시점 LLM 건당 환산원가는 "확정 불가".** 한도/구독료가 들어오면 즉시 산출한다. 예시 감만:

> *예시(가정):* 구독료가 월 $20이고 한도가 "월 토큰 10M"이라면 → 10M ÷ 600K = **약 16건/월** → 건당 약 $1.25(1,750원). 한도가 "월 30M 토큰"이면 → 약 50건/월 → 건당 약 $0.40(560원).
> **즉 한도가 어디냐에 따라 LLM 건당원가가 3~4배 출렁인다.** 이게 안 잡히면 마진을 못 박는다.

### 2-3. ⚠️ 더 큰 함정 — provider 우선순위 모순 (확인 필요)

`.env.local`에 **`ANTHROPIC_API_KEY`가 SET 되어 있다.** 그런데:
- `resolveProvider()` 우선순위: **ANTHROPIC > GEMINI > OPENAI > ZAI** → anthropic이 1순위로 선택됨
- `anthropicAdapter.complete()`는 **미구현(throw)** — 메모리(llm-provider-abstraction)·코드 둘 다 "뼈대만, 키 있으면 throw" 확인

→ **현재 .env 그대로면 모든 LLM 호출이 anthropic으로 라우팅돼 throw → mock fallback** 가능성. "z.ai 정액이 메인"이라는 전제(2026-06-24 메모리, 당시 ANTHROPIC 키 없었음)와 **모순**.

**원가 영향이 3갈래로 갈린다 → 도현 확인 2 (최우선):**
1. anthropic adapter가 그새 구현됐다면 → **LLM이 Claude 종량 API로 전환** = 원가 구조 완전히 달라짐(정액→종량, 건당 수천 원대 가능)
2. 여전히 throw라면 → 실제 문서가 mock(가짜)으로 생성되는 **품질 사고** (원가 이전에 제품 문제)
3. 운영 .env는 ANTHROPIC 키를 빼고 z.ai만 둔다면 → 메모리대로 z.ai 정액 (2-2 공식 적용)

→ **어느 쪽이 진짜 운영 구성인지 확정 전엔 LLM 원가를 못 박는다.**

## 3. 스토리지 (Supabase) 단가

녹음 파일은 STT 변환용 **임시사본만 올리고 변환 후 삭제**(메모리 stt-storage-architecture: finally에서 storage.delete). 즉 **누적 저장은 거의 없다.** 단 DB(meetings: 텍스트/문서)는 사용자별로 쌓인다.

| 항목 | 값 | 출처 |
|------|-----|------|
| Supabase 무료 티어 | 총 **1GB** 스토리지 / 파일당 50MB / DB 포함 | 실데이터: 메모리, Supabase 공식 무료 |
| 녹음 1건(20분) 크기 | **약 9.6MB** | 추정: webm/Opus, 브라우저 기본 ≈64kbps → 64kbps×1200초/8 ≈ 9.6MB. (비트레이트 미명시, useRecorder.ts에 설정 없음 → 보수 추정. **나루 확인**) |
| 녹음 보관 | **변환 후 삭제** (임시) | 실데이터: 메모리 |
| 문서 1세트(텍스트) DB | 약 100~300KB | 추정: 14종 마크다운 텍스트 |

### 무료 한도로 몇 건?

- **녹음(임시):** 동시 변환 중인 파일만 점유. 9.6MB 기준 1GB ÷ 9.6MB ≈ **순간 100건 동시**까진 무료. 삭제되므로 누적 압박 없음.
- **DB(누적):** 1GB ÷ 0.2MB ≈ **약 5,000건의 문서세트**까지 무료. 베타 규모(수십~수백 사용자)에선 **여유.**
- **초과 시 단가:** Supabase Pro $25/월(8GB 포함, 추가 GB당 과금). 베타에선 **도달 안 함 → 사실상 0원/건.**

> **결론: 스토리지는 베타 변동비에서 ~0원.** 단 (a)녹음을 영구보관으로 정책 바꾸면(원본 누적) 9.6MB/건이 빠르게 쌓여 약 100건/사용자에서 1GB 압박 → 그땐 재계산. (b)비트레이트 추정이 틀리면 크기 비례 변동.

## 4. 호스팅 (Vercel) 단가

문서생성이 20분 길지만, Vercel **Fluid Compute는 LLM 호출 대기(I/O) 시간엔 Active CPU 과금을 안 한다**(2026 공식). 20분의 대부분이 z.ai 응답 대기 = I/O라 **CPU 과금은 작다.**

| 항목 | 값 | 출처 |
|------|-----|------|
| Hobby(무료) | Active CPU 4시간/월 + Provisioned Memory 360 GB-hr/월 + 함수 100만콜/월 | 실데이터: Vercel 공식 2026 |
| Pro | $20/seat/월 (+$20 크레딧), Active CPU 초과 $0.128/hr | 실데이터: Vercel 공식 2026 |
| 함수 maxDuration | 300초(개별 문서) / PRD 라우트 800초 | 실데이터: 코드 maxDuration |
| 회의 1건 Active CPU 소비 | **추정 적음**(대기=I/O 제외) | 추정 — **도현 실측 확인(확인 4)** |

> **결론: 호스팅도 건당 변동비는 작다(추정 ~0원).** 단 (a)Hobby는 maxDuration 제약/상업이용 약관 이슈 → 베타라도 **Pro($20/월 고정비)** 필요 가능성. 이건 건당 변동비가 아니라 **월 고정비**로 잡아야 함. (b)Active CPU 실측치 없으면 헤비유저 다수 시 4시간/월 한도 돌파 여부 미지 → 도현 로그 확인.

## 5. 회의 1건 처리 총원가 — 시나리오별 한 줄 결론

> 변동비만 합산(고정비 Vercel Pro $20/월·Supabase 누적은 별도). LLM은 **2-2/2-3 미확정**이라 범위로 표기.

| 비용 항목 | 시나리오 A (브라우저STT) | 시나리오 B (Whisper) |
|-----------|------------------------|---------------------|
| STT | 0원 | 168원 |
| LLM (z.ai 정액 환산) | **? (560~1,750원 추정범위)** | **? (560~1,750원 추정범위)** |
| 스토리지 | ~0원 | ~0원 |
| 호스팅(변동) | ~0원 | ~0원 |
| **합계** | **약 560~1,750원 (LLM 확정 전)** | **약 730~1,920원 (LLM 확정 전)** |

### 한 줄 결론 (2개 시나리오)

- **시나리오 A (브라우저STT 무료):** 회의 1건 변동원가 ≈ **LLM 환산원가 그 자체** (STT·스토리지·호스팅 ~0). z.ai 한도가 들어오면 **560~1,750원** 사이에서 확정. STT를 무료로 돌리는 한, **원가의 99%가 LLM 한 덩어리**다.
- **시나리오 B (Whisper 종량):** 회의 1건 변동원가 ≈ **LLM + 약 168원**. STT가 명시적 종량이 되면 회의 길이에 정비례(20분→40분이면 STT 336원). 그래도 여전히 **LLM이 원가의 주력.**

> **핵심 한 줄:** **이 사업의 변동원가는 사실상 z.ai LLM 한 덩어리다.** STT는 무료(A)거나 길이비례 소액(B), 스토리지·호스팅은 베타에선 ~0. 따라서 **마진을 좌우하는 단 하나의 미지수 = "z.ai 정액 한도와 월 구독료"**. 이게 안 잡히면 요금제 마진을 못 박는다(2-2). 더 급한 건 provider 모순(2-3) — 잘못 라우팅되면 정액이 종량으로 둔갑하거나 mock이 생성된다.

| 가정 | 기준값 | 틀리면 | 영향 |
|------|--------|--------|------|
| 회의 길이 | 20분 | 40분 | STT(B)만 2배(336원). LLM은 토큰 소폭↑ |
| 실사용 문서 수 | 14종 | 서연 H2대로 2~4종만 | LLM 콜이 53→약 10~20콜로 급감 → **건당 LLM원가 절반 이하 가능**. 요금제를 "문서 종류 수"로 가르면 이게 직접 마진이 됨 |
| z.ai 월 한도 | 미상 | 작으면(타이트) | 건당원가↑, 헤비유저가 한도 터뜨려 **무제한 요금제 = 적자**(나의 옛 교훈 재현) |
| provider | z.ai 정액 | Claude 종량으로 전환됨 | 정액 전제 붕괴 → 건당 수천 원대 가능, **전면 재계산** |
| 녹음 영구보관 | 임시삭제 | 영구보관 정책 | 스토리지 0원→누적(9.6MB/건), 약 100건/사용자에서 1GB 돌파 |

## 7. 도현/나루에게 확인할 것 (우선순위)

1. **(도현·최우선) provider 운영 구성:** 운영 .env에 ANTHROPIC_API_KEY가 실제로 들어가나? anthropicAdapter는 구현됐나(아직 throw인가)? → LLM이 z.ai 정액인지 / Claude 종량인지 / mock인지 결정. **이게 안 잡히면 LLM 원가 전체가 미정.**
2. **(도현) z.ai 코딩플랜 실제 한도·구독료:** 월 요금($?), 월 한도(요청 수 / 토큰 / 일 prompt 수 / 동시성). → 2-2 공식의 건당 환산원가 확정용.
3. **(도현) 회의 1건당 실제 토큰 소비 로그:** 콜 53회 추정은 코드로 셌으나, 입출력 토큰 합계 실측치(평균/최대) 필요. → 한도 소진율 정확화.
4. **(도현) Vercel 운영 플랜·Active CPU 실측:** 현재 Hobby인지 Pro인지(상업베타 약관), 회의 1건당 Active CPU 실측. → 호스팅이 고정비($20)인지 변동비인지.
5. **(도현) STT 운영 분기:** 베타 운영은 브라우저STT(무료)로 가나, Whisper(종량)로 가나? .env에 OPENAI 키 SET이면 종량인데, 의도된 건가? gpt-4o-mini-transcribe($0.003)로 절반 절감 가능한가?
6. **(나루) 녹음 비트레이트:** useRecorder.ts에 비트레이트 미명시 → 실제 webm/Opus 비트레이트(파일 크기/건). 9.6MB 추정 검증용.
