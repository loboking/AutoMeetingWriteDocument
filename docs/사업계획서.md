## 2. 문제 정의 (Problem)

소프트웨어 프로젝트의 출발점은 회의지만, 회의 결과를 **기획 문서로 옮기는 과정**에 막대한 시간이 소모된다.

- 회의에서 결정된 내용이 **문서화되지 않아** 개발 단계에서 누락·재논의 발생
- PRD·화면설계·DB설계·API명세 등 **산출물 작성에 기획자 며칠** 소요
- 소규모 팀·1인 개발자는 **기획 인력 자체가 부재**하여 문서 없이 개발 강행 → 품질·일정 리스크
- 기존 회의록 도구는 **요약까지만** 제공, 실제 개발에 쓰는 산출물로 이어지지 않음

> **핵심 문제**: "회의는 끝났는데, 문서는 아무도 안 만든다."

---

## 3. 솔루션 (Solution)

회의 녹음 한 번으로 **개발 실무에 바로 쓰는 기획 문서 14종**을 자동 생성한다.

![요약 화면](../public/landing/03-summary.png)

### 처리 흐름
1. **입력** — 브라우저 녹음 또는 녹음 파일 업로드
2. **회의록** — STT로 음성을 텍스트화하고, AI가 개요·핵심 논의·결정 사항·할 일로 구조화
3. **문서 생성** — 요약을 바탕으로 문서 14종을 의존성 순서에 따라 자동 작성
4. **활용** — Markdown·Word·Excel·PowerPoint로 내보내 팀과 공유

### 자동 생성 문서 14종
PRD · 시나리오 정의서 · 기능목록 · 화면목록 · IA · DB설계 · API명세 · 테스트계획 · 플로우차트 · 스토리보드 · 와이어프레임 · WBS · 테스트케이스 · 배포가이드

## 5. 시장 분석 (Market)

- **대상 시장**: 국내 소프트웨어 개발사·SI/SM·디지털 에이전시·스타트업
- **확장 시장**: 기획 산출물이 필요한 모든 프로젝트 조직 (공공 SI 포함)
- **시장 동력**:
  - 생성형 AI 도입 가속으로 "문서 자동화" 수요 급증
  - 개발 인력 대비 기획 인력 부족 → 자동화 니즈 구조적 존재
  - 음성 인식·LLM 단가 하락으로 단위 처리비용 지속 감소

| 모델 | 내용 |
|------|------|
| 기본 | 구독형 SaaS (월/연 단위) |
| 과금 기준 | 회의 처리 건수 / 생성 문서 수 / 팀 좌석 |
| 확장 | 기업용 온프레미스·API 제공, 기존 협업툴 연동 |

## 7. 개발 현황 (Traction)

- **작동하는 웹 애플리케이션** 구현 완료 (녹음 → 회의록 → 문서 생성 전 과정)
- 문서 14종 생성 및 4종 포맷(DOCX·XLSX·PPTX·MD) 내보내기 동작
- 사용자 계정 기반 데이터 저장·동기화 구현
- 문서 간 의존성 전파, 시각화(Mermaid) 렌더링 내장
